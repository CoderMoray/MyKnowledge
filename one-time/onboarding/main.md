# MyKnowledge 首次引导

> ⚠️ **AI 助手注意**：此文件仅在检测到 skill-state.yaml 不存在时加载。
>
> **🔒 硬性规则（不可跳过）**：
> - **必须按序完成全部 5 个步骤（1→2→3→4→5），缺一不可**
> - 每步骤完成后，必须获得用户明确确认，才能进入下一步骤
> - **只有在步骤 5 全部完成后，才允许写入 `skill-state.yaml`**
> - 禁止在任何步骤中途跳过、合并、或"智能推断"用户意图
> - 如果用户在任何步骤中说了无关内容，礼貌提醒先完成引导

---

## 引导流程

> 每步骤标注 `@阻塞性`（必须完成才能继续）或 `@自动`（可自动执行但需展示结果）

### 步骤 1：欢迎语 `@阻塞性`

```
👋 欢迎使用 MyKnowledge！

我是你的智能知识库助手，可以帮你：
📁 自动整理项目资料
📝 记录要做的事情
📊 跟踪任务进度
🤖 自动帮你做记录（可选）

让我花 1 分钟帮你设置一下...
```

**完成条件**：展示欢迎语后，等待用户回复（任何回复即可，表示用户已看到）。

---

### 步骤 2：平台确认 `@阻塞性`

> `@auto-detect`：如果可以通过当前运行环境自动确定平台，则自动填入并展示给用户确认，无需让用户手动选择。

```
你使用什么 AI 助手？
[CodeBuddy] [WorkBuddy] [OpenClaw] [Claude] [其他]
```

**自动检测规则**：
- 当前环境有 `CODEBUDDY_*` 环境变量 → 自动选 `CodeBuddy`
- 当前环境有 `WORKBUDDY_*` 环境变量 → 自动选 `WorkBuddy`
- 当前环境有 `OPENCLAW_*` 环境变量 → 自动选 `OpenClaw`
- 在 Claude 对话中 → 自动选 `Claude`
- 无法确定 → 展示选项让用户选择

**完成条件**：平台已确定（自动或手动），展示确认信息"已检测到平台：{platform}"。

---

### 步骤 3：自动记录设置 `@阻塞性`

```
是否开启「自动记录」？

开启后，当你讨论复杂任务时，我会自动帮你创建记录。

[开启] - 自动记录（推荐）
[关闭] - 手动控制

💡 随时可以说"开启/关闭自动记录"修改
```

**完成条件**：用户明确选择"开启"或"关闭"，展示确认信息。

---

### 步骤 4：保存配置 `@自动`

> **前置条件**：步骤 1、2、3 全部完成且确认。

创建 `~/.myknowledge/config/skill-state.yaml`：

```yaml
initialized: true
platform: "{user_platform}"
auto_record: true
onboarding_completed: true
first_use: "{date}"
version: "1.2.3"
```

创建 `~/.myknowledge/config/install-source`：
> **详细检测逻辑**：参考 `one-time/setup/install-source.md`

创建 `~/.myknowledge/config/projects.yaml`（空项目目录，后续自动维护）：

```yaml
projects: []
```

**完成条件**：3 个文件全部写入成功。

---

### 步骤 5：结束语 `@阻塞性`

> **前置校验**：执行前必须确认 `skill-state.yaml` 已写入且 `onboarding_completed: true`。若未写入，回到步骤 4。

```
🎉 设置完成！记住这 3 句话：
• "创建知识库" - 开始新项目
• "项目进展如何？" - 查看状态
• "继续之前的项目" - 恢复工作

✅ 引导已完成，下次不再显示
```

**完成条件**：展示结束语，用户确认收到。

---

## 引导完成后的硬性禁止

引导完成后（`onboarding_completed: true`），**禁止**：
- ❌ 重新加载本文件（除非用户说强制重新引导的关键词）
- ❌ 在正常使用中重新执行任何引导步骤
- ❌ 覆盖已有的 `skill-state.yaml`

---

## 强制重新引导

如果用户说以下指令，删除 skill-state.yaml 并重新引导：
- "重新初始化 MyKnowledge"
- "重置 MyKnowledge"
- "显示引导"
