# MyKnowledge 命令速查

> **AI 加载触发**：当用户询问"你能做什么"、"怎么用"、"有哪些命令"、
> 或对某项操作表达不确定时，加载本模块。
>
> 加载本模块后，**使用下方表格作为权威回答来源**，不要凭记忆回答。

---

## 核心原则

1. **固定命令优先**：下面的"固定命令"表是用户最不容易出错的方式
2. **意图识别辅助**：同义说法会被自动识别为同一命令
3. **模糊时主动确认**：拿不准时不要猜，先问"你是想：创建 / 查看 / 更新 / 归档？"

---

## 固定命令速查表

| 用户说 | 你要做 | 等价说法 |
|--------|--------|----------|
| **"创建知识库"** | 询问类型（全局/项目）并初始化 | "新建知识库"、"开始新项目" |
| **"新建需求 xxx"** | 创建 REQ-YYYYMMDD-XXX 目录与文档 | "添加任务 xxx"、"记录 xxx" |
| **"REQ-001 完成"** | 状态转 Done，追加归档时间戳 | "REQ-001 搞定了"、"完结 REQ-001" |
| **"REQ-001 进行中"** | 状态转 In Progress | "开始 REQ-001"、"继续 REQ-001" |
| **"REQ-001 待审核"** | 状态转 Review | "REQ-001 等你审" |
| **"REQ-001 取消"** | 状态转 Cancelled | "不做了 REQ-001" |
| **"查看 REQ-001"** | 读取并展示该需求文档 | "REQ-001 是什么"、"REQ-001 进展" |
| **"项目进展如何"** | 汇总 PROJECT-STATUS.md | "现在到哪了"、"状态怎么样" |
| **"继续 xxx"** | 加载 PROJECT-STATUS 并恢复上下文 | "接着做 xxx"、"回到 xxx 项目" |
| **"搜索 xxx"** | 跨需求/文档关键词搜索 | "找一下 xxx"、"有没有 xxx"（v1.2+） |
| **"开启自动记录"** | 设置 skill-state.yaml 中 auto_record=true | "启用静默模式" |
| **"关闭自动记录"** | 设置 skill-state.yaml 中 auto_record=false | "别自动记录" |
| **"重新初始化"** | 删除 state 并触发 onboarding | "重置 MyKnowledge" |

## 意图识别（同义表达映射）

当用户说的话**不在固定命令表里**，但意图明显时，按下方映射处理：

| 用户意图 | 触发词示例 | 对应操作 |
|----------|-----------|----------|
| 创建类 | "记录"、"加个"、"写下"、"建一个"、"我要做" | → 新建需求 |
| 更新类 | "改"、"更新"、"编辑"、"备注"、"补充" | → 更新现有 REQ |
| 完成类 | "完事"、"OK 了"、"好了"、"交付了"、"结了" | → 状态转 Done |
| 查看类 | "看"、"显示"、"列出来"、"现在怎样"、"多少个" | → 读取状态 |
| 归档类 | "归档"、"收起来"、"放 archive" | → 移到 archive/ |
| 搜索类 | "搜"、"找"、"有没有"、"在哪" | → 关键词检索 |

## 模糊时的确认话术

**不要瞎猜**。如果用户的话不在以上任何一类，按下方模板追问：

```
🤔 你的意思是？

[创建]   创建一个新的 REQ 记录
[查看]   查看某需求的当前状态
[更新]   更新某需求的内容或状态
[归档]   把已完成的需求移到 archive/

或者你可以直接说："新建需求 xxx" / "查看 REQ-001" / "项目进展如何"
```

## 能力范围速览（避免越界）

**本 Skill 擅长**：
- 创建和管理本地知识库目录结构
- 跟踪 REQ-* 需求的全生命周期
- 自动把对话追加到当前活跃需求
- 在 PROJECT-STATUS.md 维护项目快照

**本 Skill 不做**（不要尝试）：
- ❌ 联网/同步到云端
- ❌ 多用户协作
- ❌ 修改 ~/.myknowledge/ 之外的文件
- ❌ 执行任意 shell 命令
- ❌ 富文本/所见即所得编辑

详见 README.md 的"⚠️ 能力边界"小节。
