# 错误处理模块

> 💡 **恢复极其简单**：遇到任何问题，说一句"重新初始化 MyKnowledge"即可恢复——
> 无需手动操作任何文件，不会丢失知识库数据。Skill 不自动重试，是为了确保每次修改都经过你确认。

当遇到错误时，按本模块指引处理。

---

## 错误分类与解决

### 1. 创建失败

| 错误类型 | 表现 | 可能原因 | 解决 |
|----------|------|----------|------|
| 知识库创建失败 | 无法创建目录/文件 | 当前文件夹没有写入权限 | 换个有写入权限的文件夹试试 |
| 知识库创建失败 | 无法创建目录/文件 | 路径包含非法字符 | 文件夹名里有特殊符号（如冒号、星号），去掉后重试 |
| 知识库创建失败 | 无法创建目录/文件 | 磁盘空间不足 | 磁盘快满了，清理一些空间再试 |
| 文件操作卡住 | 写入失败、文件被占用 | 文件被其他程序占用 | 关闭相关程序（编辑器、终端等）后重试，说"重新初始化"可恢复 |
| 需求创建失败 | 无法创建需求目录 | 知识库未初始化 | 还没创建知识库呢，先对我说"创建知识库" |
| 需求创建失败 | 无法创建需求目录 | 需求 ID 冲突 | 同名需求已经存在，换个名字试试 |

### 2. 状态更新失败

| 错误类型 | 表现 | 可能原因 | 解决 |
|----------|------|----------|------|
| 状态更新失败 | 无法修改状态 | 文件损坏 | 文件可能损坏了，说"重新初始化"一键修复 |
| 状态更新失败 | 无法修改状态 | 格式错误 | 文档格式异常，说"重新初始化"一键修复 |

### 3. 配置相关

| 错误类型 | 表现 | 可能原因 | 解决 |
|----------|------|----------|------|
| 配置损坏 | 无法读取 skill-state.yaml | 文件被手动编辑 | 配置文件可能被手动改坏了，说"重新初始化"即可恢复 |
| 平台检测错误 | 无法识别平台 | 目录不存在 | 告诉我你用什么平台或软件（CodeBuddy / WorkBuddy / OpenClaw / Claude / 你使用的软件名称） |
| 安装源检测失败 | 无法确定安装方式 | 多平台混用 | 告诉我你是怎么安装的（SkillHub 还是 GitHub 或者是其他方法？） |

### 4. 静默模式相关

| 错误类型 | 表现 | 可能原因 | 解决 |
|----------|------|----------|------|
| 检测偏好调整 | 触发频率不符合预期 | 阈值需微调 | 灵敏度可以调整，对我说"调整检测灵敏度"即可 |

### 5. 项目目录不存在

| 错误类型 | 表现 | 可能原因 | 解决 |
|----------|------|----------|------|
| 项目目录不存在 | 读取 PROJECT-STATUS.md 失败 | 用户手动删除了项目文件夹 | 该项目目录已被删除，我帮你从项目列表中移除。下次创建同名项目时会重新记录 |

> **处理流程**：读取 PROJECT-STATUS.md 失败时 → 从 projects.yaml 中移除该条目 → 提示用户已清理 → 询问是否切换到其他项目。

### 6. 未列举错误（兜底）

> **重要**：本表未覆盖的错误，请按下方流程处理。

| 错误类型 | 表现 | 解决 |
|----------|------|------|
| 未知错误 | 表格中没列 | 1. 根据错误消息推断原因；2. 从 FAQ.md 中查找匹配答案并直接告诉用户；3. 列出最可能的原因（权限/文件占用/路径/版本），引导用户逐一排查；4. 仍无法解决→帮用户执行"重新初始化" |
| 平台不支持 | AI 报"不识别" | 告诉我你用什么软件（CodeBuddy / WorkBuddy / OpenClaw / Claude 或其他） |
| 文件夹权限 | 无法读写 ~/.myknowledge/ | 知识库文件夹没有读写权限，我会告诉你需要执行的操作 |
| 路径错误 | 找不到知识库 | 换个项目目录试试，或对我说"项目进展如何"重新定位 |

> 💡 **快速自愈**：遇到任何未列举错误，说"重新初始化 MyKnowledge"即可恢复。不会丢失知识库数据，所有文件（~/.myknowledge/）保留不变。

## 快速诊断清单

遇到问题时，先检查：

```
□ 是不是在正确的项目文件夹里？
□ 文件夹能不能写入（不是只读的）？
□ 知识库是不是已经创建过了？
□ 需求的名字格式对吗？（让 AI 自动生成就好，不用自己起名）
□ 配置文件是不是被手动改过？
```

---

## 配置恢复

如果 skill-state.yaml 损坏：

```
1. 备份现有配置：
   cp ~/.myknowledge/config/skill-state.yaml ~/.myknowledge/config/skill-state.yaml.bak

2. 删除损坏文件：
   rm ~/.myknowledge/config/skill-state.yaml
   rm ~/.myknowledge/config/install-source

3. 重新触发 Skill，执行首次引导重建配置
```

## 错误响应格式

遇到错误时，按以下格式响应。**关键原则：直接给用户解决方案，不要让用户自己去查文档。**

```
⚠️ 遇到问题：{通俗的错误描述}

原因：{一句话说清楚为什么}

解决：{直接告诉用户怎么做，用日常语言}
（恢复极其简单——说"重新初始化"即可，不丢数据，无需手动操作文件）
```

### AI 自检：回复前确认
- [ ] 用户看完回复后还需要去别的地方查吗？如果需要→把答案直接写进回复
- [ ] 用了技术术语吗？如果用了→换成日常说法
- [ ] 用户知道下一步做什么吗？如果不知道→给出明确的一步操作
