# 静默模式说明

## 什么是静默模式？

静默模式让 MyKnowledge 在**用户无感知**的情况下自动工作。

当用户提出复杂任务时，AI 自动：
1. 创建知识库（如不存在）
2. 创建需求记录
3. 跟踪任务进度

## 两种静默模式

### 模式 1：Hook 静默（OpenClaw 专属）

**原理**：通过 OpenClaw Hook 事件驱动

**流程**：
```
用户发送消息
    ↓
OpenClaw 触发 message:received 事件
    ↓
MyKnowledge Hook 自动执行
    ↓
分析任务复杂度
    ↓
自动创建知识库和记录需求
    ↓
（用户完全无感知）
```

**特点**：
- ✅ 完全无感知
- ✅ 100% 自动
- ⚠️ 仅 OpenClaw 支持
- ⚠️ 需要手动启用 Hook

### 模式 2：意图识别静默（通用）

**原理**：AI 通过 Prompt 判断自动执行

**流程**：
```
用户发送消息
    ↓
AI 分析消息内容
    ↓
检测到复杂任务
    ↓
自动创建知识库和记录需求
    ↓
告知用户："已自动记录需求 REQ-XXX"
```

**特点**：
- ✅ 所有平台支持
- ✅ 无需额外配置
- ⚠️ 轻微感知（AI 会告知）
- ⚠️ 依赖 AI 判断准确性

## 复杂任务检测

### 检测规则

**关键词匹配**（满足 2 个及以上）：
- 分析、统计、挖掘
- 开发、设计、调研
- 整理、清洗、项目

**任务特征**：
- 多步骤操作
- 需要长期跟踪
- 涉及数据或文档

### 示例

| 用户输入 | 检测结果 | 行为 |
|----------|----------|------|
| "帮我算 1+1" | ❌ 简单任务 | 不创建知识库 |
| "分析销售数据" | ✅ 复杂任务 | 自动创建知识库 |
| "开发一个工具" | ✅ 复杂任务 | 自动创建知识库 |
| "整理我的笔记" | ✅ 复杂任务 | 自动创建知识库 |

## 用户控制

### 全局开关

在 `settings.yaml` 中：

```yaml
features:
  silent_mode:
    enabled: true   # 启用
    # enabled: false  # 禁用
```

### 单次禁用

用户可以说：
- "这次不要记录"
- "不要创建知识库"
- "简单处理"

### 查看自动记录

```
用户：查看自动记录的内容
AI：您在本次会话中自动创建了：
    - 知识库：~/MyKnowledge/global/
    - 需求：REQ-20260608-001 "分析销售数据"
```

## 存储位置

### 自动创建的默认位置

**全局知识库**：
```
~/MyKnowledge/global/
```

**项目知识库**（如果当前在项目目录）：
```
{project-path}/.myknowledge/
```

### 修改默认位置

在首次引导时选择，或修改 `settings.yaml`：

```yaml
storage:
  global:
    path: "~/MyKnowledge/global"  # 修改为自定义路径
```

## 隐私说明

- 所有数据存储在**本地**
- 不上传到任何服务器
- 用户完全掌控自己的数据

## 关闭静默模式

如果希望完全手动控制：

1. 对 OpenClaw：禁用 Hook
   ```bash
   openclaw hooks disable myknowledge
   ```

2. 对其他平台：修改 `settings.yaml`
   ```yaml
   features:
     silent_mode:
       enabled: false
   ```

3. 或删除 `skill-state.yaml` 重新配置
