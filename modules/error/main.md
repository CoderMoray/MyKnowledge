# 错误处理模块

当遇到错误时，按本模块指引处理。

---

## 错误分类与解决

### 1. 创建失败

| 错误类型 | 表现 | 可能原因 | 解决 |
|----------|------|----------|------|
| 知识库创建失败 | 无法创建目录/文件 | 权限不足 | 检查目录权限，切换到有权限的目录 |
| 知识库创建失败 | 无法创建目录/文件 | 路径包含非法字符 | 检查路径是否包含 `<>:"|?*` |
| 知识库创建失败 | 无法创建目录/文件 | 磁盘空间不足 | 清理磁盘空间 |
| 需求创建失败 | 无法创建需求目录 | 知识库未初始化 | 先创建知识库 |
| 需求创建失败 | 无法创建需求目录 | 需求 ID 冲突 | 使用新的 ID |

### 2. 状态更新失败

| 错误类型 | 表现 | 可能原因 | 解决 |
|----------|------|----------|------|
| 状态更新失败 | 无法修改状态 | 文件损坏 | 备份后删除重建 |
| 状态更新失败 | 无法修改状态 | 格式错误 | 检查 PROJECT-STATUS.md 格式 |

### 3. 配置相关

| 错误类型 | 表现 | 可能原因 | 解决 |
|----------|------|----------|------|
| 配置损坏 | 无法读取 skill-state.yaml | 文件被手动编辑 | 删除后重新引导 |
| 平台检测错误 | 无法识别平台 | 目录不存在 | 手动指定平台 |
| 安装源检测失败 | 无法确定安装方式 | 多平台混用 | 用户主动告知 |

### 4. 静默模式相关

| 错误类型 | 表现 | 可能原因 | 解决 |
|----------|------|----------|------|
| 静默误触发 | 简单对话触发创建 | 关键词太宽泛 | 关闭 auto_record 或提高阈值 |
| 静默不触发 | 复杂任务未检测 | 关键词不匹配 | 检查 settings.yaml 配置 |

---

## 快速诊断清单

遇到问题时，先检查：

```
□ 是否在正确的项目目录？
□ 是否有写入权限？
□ 知识库是否已初始化？
□ 需求 ID 格式是否正确（REQ-YYYYMMDD-XXX）？
□ 配置文件是否损坏？
```

---

## 配置恢复

如果 skill-state.yaml 损坏：

```
1. 备份现有配置：
   cp ~/.myknowledge/config/skill-state.yaml ~/.myknowledge/config/skill-state.yaml.bak

2. 删除损坏文件：
   rm ~/.myknowledge/config/skill-state.yaml
   rm ~/.myknowledge/config/install-source

3. 重新触发 Skill，执行首次引导重建配置
```

## 错误响应格式

遇到错误时，按以下格式响应：

```
⚠️ 遇到问题：{错误描述}

可能原因：{原因1}、{原因2}

解决方法：
1. {解决步骤1}
2. {解决步骤2}

如果问题持续，请说"重新初始化 MyKnowledge"重置配置。
```
